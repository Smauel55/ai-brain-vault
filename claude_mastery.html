SECTION 01
THE 3 TOOLS

Each tool has a distinct job. Using the right one is the skill that separates power users from casual ones.

Claude Chat
Your thinking partner — claude.ai or the mobile app
  • Brainstorming & strategy
  • Research, summaries, analysis
  • Writing: emails, pitch decks, essays
  • Learning new concepts fast
  • Feedback & decision-making
  • Planning, advice, quick Q&A
Cowork
A desktop agent that operates your computer
  • Organize files & folders
  • Batch rename or sort files
  • Automate repetitive file tasks
  • Reformat spreadsheets or docs
  • Manage project assets in bulk
  • Multi-step desktop workflows
Claude Code
A command-line tool that builds real software
  • Build apps, sites, and tools
  • Vibe code without knowing syntax
  • Automate complex multi-step logic
  • Modify and debug codebases
  • Create APIs, scripts, data pipelines
  • Anything requiring actual software
Mental model: Chat is the co-founder you text at 2am. Cowork is the ops assistant who organizes your files while you sleep. Code is the engineering team who ships software from a plain English description.

SECTION 03
30-DAY LEARNING PLAN

Structured for zero coding experience. Complete this before Northwestern and you'll be ahead of 99% of your classmates on day one.

1
Master Chat
DAYS 1–5
Open claude.ai and just talk. Create a Project called 'Northwestern Prep' — give Claude context about your goals; it remembers across every conversation in that project. Turn on Web Search. Upload a PDF and ask Claude to analyze it. Ask it to create an Artifact (chart, calculator, table) directly in chat.
Project: Write a full business plan for a startup idea. Ask Claude to poke holes in your plan, then help you rebuild it stronger.
2
Explore Cowork
DAYS 6–9
Download the Cowork desktop app. First task: organize your Downloads folder by file type. Try a batch rename. Then try a multi-step workflow: 'Find all PDFs from the last 30 days and move them to a folder called Recent Research.' Learn its limits — great for files, not for building software.
Project: Use Cowork to create a clean file structure for all your college prep materials and project assets.
3
Vibe Code Your First App
DAYS 10–18
Install Node.js from nodejs.org. Run: npm install -g @anthropic-ai/claude-code — then open Terminal in a project folder and type claude. Describe what you want built in plain English. Ask Claude Code to teach you git basics — it's like saving your game in Minecraft. You never touch the code directly.
Project: Build a personal homepage showing your goals, a reading list, and links to your projects. Then ask Claude Code to add a dark mode toggle.
4
Combine the Tools
DAYS 19–25
Start a real startup project. Use Chat to brainstorm and plan → Cowork to organize assets → Code to build the product. Practice the handoff deliberately. This is the full founder workflow.
Project: Pick a real problem you have. Chat to define it, Cowork to manage files, Code to build the MVP.
5
Go Deep
DAYS 26–30+
Learn MCP connectors (Claude Code can connect to GitHub, Slack, Notion). Explore the Claude API. Study prompt engineering at docs.claude.com. Start an 'AI experiments' journal — when something works brilliantly or fails, write it down.
Project: Build something genuinely useful. This becomes your first AI portfolio piece for Northwestern clubs and interviews.
SECTION 04
VIBE CODING

Building real software by describing what you want in plain English. No syntax. No confusion.

01
Set up
Install Node.js from nodejs.org. Run: npm install -g @anthropic-ai/claude-code. Create a folder, open Terminal inside it, type: claude
02
Describe, don't code
Write what you want in plain English. 'Build me a webpage where I input a startup idea and get back a list of risks.' That's a complete prompt.
03
Review intent
Claude Code tells you every step it's taking. You don't need to understand the code — understand the goal of each step. Ask 'what does this do?' freely.
04
Iterate in English
'Make the button bigger.' 'Add a dark mode.' 'Show a loading spinner on submit.' Every change is a conversation, not a code edit.
05
When things break
'When I click the button, nothing happens.' That's enough to diagnose and fix. Describe the symptom, not the technical cause.
06
Ship it
Say 'deploy this to Vercel' or 'push to GitHub.' Claude Code walks you through every step including account setup. Your app goes live.
The mindset shift: You are the product manager and designer. Claude Code is your engineer. Describe the experience you want — not how to build it. The better your description, the better the output.

SECTION 06
THE ART OF PROMPTING

The single highest-leverage AI skill. Weak prompts get generic output. Precise prompts get output that feels like magic.

PART 1
Role / Context
Tell Claude who it is. "You are a YC partner who has reviewed 10,000 pitches."
PART 2
Specific Task
Not "help with my pitch" — instead "identify the three weakest assumptions in my value proposition."
PART 3
Output Format
Specify how you want the answer. "A numbered list." "Under 200 words." "A pros/cons table."
PART 4
Constraints
"No technical jargon." "Assume I'm a beginner." "Focus only on B2B." Tell it what to avoid.
Weak vs. Strong Examples
❌ Weak
Help me with my startup idea.
✅ Strong
You're a YC partner who has reviewed 10,000 pitches. My idea is [X]. Give me 3 bullets on why startups like mine typically fail, and 3 on what would make an investor immediately excited. Be direct, don't sugarcoat.
❌ Weak
Write me a cold email.
✅ Strong
Write a cold outreach email to the CTO of a Series A startup. I want to offer to build a free automation tool for their team to show my skills. Keep it under 100 words, human and non-salesy. Leave a placeholder for one specific company observation I'll fill in. Write three versions with different opening hooks.
Best tip: Treat Claude like the smartest person you've ever worked with, not a search engine. Give it context, constraints, and tell it what good looks like. When stuck, ask: "What information would help you give me a better answer?" That question alone will save you hours.

Full prompt engineering guide: docs.claude.com/en/docs/build-with-claude/prompt-engineering/overview
