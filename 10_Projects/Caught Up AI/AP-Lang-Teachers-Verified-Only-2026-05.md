## Read this before using the list

**"Verified" here is weaker than the original brief assumed.** The research agent's web_fetch only accepted URLs that had already appeared in messages or earlier fetch results, so it couldn't fetch most school pages directly. The Verified emails above came from WebSearch result snippets that quoted the underlying page. That's better than a domain-pattern guess, but it isn't the same as having loaded the page yourself. Before sending anything, manually open each Source URL and confirm the email is still on the page.

**Six of the ten rows are Lane Tech accounts where the full teacher name wasn't extractable from the snippet.** The Lane Tech English department's Google Site lists teachers under usernames like `dastrom` and `kelaroche1` in the search snippets the agent could see, but the agent couldn't pull the full directory page to pair each username to a person. You can resolve this in five minutes by opening the source URL in a browser, but until you do, those rows are role-and-email-confirmed but not name-confirmed.

**Wolfe at Highland Park vs. Deerfield is unresolved.** Same person, same email, same district (Township HSD 113), but the two sub-domains (dist113.org and dhs.dist113.org) gave conflicting campus attributions. Don't address him to the wrong building.

**The honest count.** Filter strictly for "Verified email + AP Lang named in the role + full name confirmed + campus confirmed" and you have three rows: Weiler, Chandler, Wolfe (pending campus). Everything else needs a manual pass on the source URL first.
