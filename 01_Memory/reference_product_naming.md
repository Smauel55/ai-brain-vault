Naming convention for Caught Up AI, set by Samuel on 2026-06-05. Use these terms consistently in product copy, file names, and conversation.

- **Product / company:** Caught Up AI.
- **The daily unit (the PDF a teacher receives each morning):** an **Opener**. Use as a count noun: "today's Opener", "three Openers", "the Opener for June 5". Chosen because the live site already sells "AP English Language Lesson Openers Daily", so it reinforces the existing SEO and brand.
- **Each day's instance:** an **edition** (or issue).
- **The two versions:** **teacher copy** and **student copy**.
- File naming pattern: `Caught Up AI Opener - YYYY-MM-DD - <Topic> (<Register>) - Teacher copy.pdf` (and `- Student copy.pdf`). Generator: 10_Projects/Caught Up AI/Sample-Briefs/render_briefs.py.

**Avoid the word "brief" for the product.** It already means two other things in this project: the *generation brief* (the drafting prompt that writes a piece) and Samuel's *meeting brief*. Calling the product a "brief" would muddy all three.
